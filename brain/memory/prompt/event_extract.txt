请作为事件提取器，整理接收到的信息，将其汇总为记忆。
消息会以（ 时间 <发言角色> 发言内容 ）的格式出现，需要将消息结合上下文进行整理。
对于未完成的发言，可留到之后轮次，暂不进行处理。
信息应该被升维抽象为事实/关系/模式进行储存，具体储存内容如下。


【无需记忆的内容】
- 纯情绪宣泄类，缺乏具体意义的内容。
- 礼貌寒暄，社交噪声类内容。
- 即时玩梗，流行语，明显错误的反讽or玩笑信息（如：我年薪百万，我是秦始皇，等）
- 较为日常，无后续价值or会快速过期的的信息（如：要去吃饭，短暂离开很快回来，等）
- 纯复读的短期重复信息，除了第一次以外后续皆无意义。
- 结合上下文来看，未完成发言的句子。
- 在近期已经储存过的重复内容（除非该内容已经发生了变化需要修改）

【输出内容说明】
event：记忆内容的具体描述。
例："event": "张三让六个人在公屏选七个数字"

description：辅助记录事件的具体信息，
可包含：
- 强相关的聊天内容（时间 <发言角色> 发言内容）原消息
- 语气：发言人表现为自信？怀疑？不确定？或是其他
- 情景：是现实reality，还是某个小说/电影/游戏设定（如：minecraft我的世界，跑团，漫威宇宙）
- 事件可能发生所在的时间Time？地点Location？
- source消息是谁提供的？还是事实发生的被你记录的？
例："description": [
  {"text": "2026/1/2T09:36 <张三> 其实只有六个人，那这样吧"},
  {"text": "2026/1/2T09:37 <张三> 这里有1~7，七个数字"},
  {"text": "2026/1/2T09:38 <张三> 你们公屏决定"},
  {"text": "2026/1/2T09:39 <张三> 谁选哪个数字"},
  {"tone": "不确定/没想好"},
  {"context": "reality现实"}
  {"source": "来自QQ群12345678的消息记录"},
  {"importance": "只是临时选择，重要性不高"}
]


【输出格式】输出格式严格为 JSON：
{
  "event": "",
  "description": [...]
}
